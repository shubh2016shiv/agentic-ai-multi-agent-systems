# Chapter 5 — Gotchas, Edge Cases & Anti-Patterns

> **Learning chapter** — The traps and pitfalls of implementing Multi-Agent Orchestration architectures.

---

## Gotcha 1: The Parallel Agent Token Explosion

**The Scenario:** You build a fantastic Graph of Subgraphs. You have the Master Supervisor, which calls the Cardiac Department Supervisor, which fires off 4 specialized cardiac agents in parallel. You run the patient workload.
**The Failure:** "Why did this single request cost $5.00 on the OpenAI dashboard?"
**The Fix:** Every time an LLM is invoked, you pay for the input prompt. If your system prompt + patient data is 4,000 tokens long, and you trigger 10 agents simultaneously in a massive fan-out pattern, you just sent 40,000 input tokens to OpenAI in half a second. 

You must pass the `TokenManager` into `invoke_specialist` to put a strict budget limit on parallel invocations spanning large nested graphs!

## Gotcha 2: Returning unstructured data from Specialists

**The Scenario:** You delete `OrchestrationResult` because you "just want the agent's string." The Pulmonologist node returns `{"output": "Lungs are good."}`.
**The Failure:** When the Synthesizer node pulls the state `[output1, output2, output3]`, it has absolutely no idea which string belongs to which doctor. Did Cardiology say the lungs were good? 
**The Fix:** You must wrap the `output` string in an Envelope that includes the `agent_name` and `specialty`. If you don't use `OrchestrationResult`, you must use a rigid Pydantic equivalent.

## Gotcha 3: Infinite Review Loops

**The Scenario:** You try to mix the `PeerToPeer` orchestration (where specialists can talk back and forth to each other until they agree) with the `HITL` Human Review module.
**The Failure:** The Cardiology agent triggers the Pulmonology agent. The Pulmonology agent tells the Cardiology agent it needs a change. This triggers a human review. The human hits "Approve," which fires the graph back to Cardiology, which triggers Pulmonology... forever.
**The Fix:** Graph theory requires careful edge planning. If you allow cyclomatic paths (A → B → A), you MUST implement a hard `max_iterations` counter in the `State` dictionary. The Orhcestration Base Class will not save you from an infinite LangGraph recursion loop.

## Anti-Pattern 1: The Massive "God" Synthesizer

**The Scenario:** You decide that splitting the prompt up requires too many tokens, so you tell the Synthesizer to *also* do the medical assessment. "Read the patient chart, make your own assessment, and combine it with the specialists."
**The Reality:** The Synthesizer is overloaded. It forgets what the specialists said and just writes its own generic medical advice, completely ignoring the expensive insights generated by the Pulmonology agent.
**The Fix:** Keep the Synthesizer "dumb." Its system prompt should strictly be: "You are an editor. Read these three reports and format them together. Do not invent new clinical findings." The intelligence lives in the `invoke_specialist` calls.
